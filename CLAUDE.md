# 彩票数据分析项目

本仓库用于**大乐透、双色球、快乐八**的数据采集、历史分析、统计型预测参考与投注组合优化。

规则体系：`CLAUDE.md`（本文件，Claude Code 全局规则）← 与 `.cursor/rules/*.mdc`、`.cursor/agents/*.md`、`AGENTS.md` 保持一致。

---

## 合规与表述

- 彩票为随机游戏，历史统计**不能**保证未来结果。禁止「稳赚」「必中」「投资回报率」等误导表述。
- 输出预测或推荐组合时，必须声明：**娱乐目的、概率本质、过往统计仅供参考**。
- 不协助未成年人购彩；不协助规避监管或伪造开奖数据。

## 默认统计窗口

未指定期数时，三彩种频率/冷热/遗漏等主体统计**默认仅对期末尾连续 30 期**（`DEFAULT_STATS_WINDOW`）。若用其他 N 须显式写明。

快乐八在「开奖 20 码」与「选十玩法」上的语义区分须遵守，但窗口长度与 DLT/SSQ 相同。

## 数据约定

- **processed CSV 不含开奖日期列**，仅 `lottery_type` + `period_id` + 号码列。
- 分析与预测 Agent **优先**读取 `data/processed/`；若不存在则回退 `data/raw/` 或用户指定路径，并在输出中写明数据来源层级。
- **本仓库不约定存放 xlsx**。用户提供 xlsx 时须由采集 Agent 转 CSV 写入 `processed/` 并在 `manifest.json` 记录来源。
- 写入数据前执行校验：号码区间、去重、期号单调性。
- API Key / Cookie 放环境变量或 `.env`（已 `.gitignore`），禁止写入仓库明文。
- Manifest 条目按 `lottery_type` 分块（dlt/ssq/kl8），更新整文件时保留其他块。
- 导入后若发现冲突，以用户指定的权威来源为准并记录决策（可维护 `data/processed/CHANGELOG.md`）。
- 来源优先级：官方渠道 > 用户提供的可追溯文件 > 第三方交叉验证（不可作唯一权威）。

## 硬性规则（所有任务必须遵守）

### 最新期强制重算

每次「分析/预测/组号」请求，必须先以 `data/processed/*.csv` 的**最新期号**为基准重算。若归档末期小于 CSV 最新 `period_id`，必须先重算再输出。

### analysis 强制同步刷新

每次执行预测或分析任务时，必须在同一轮内同步刷新对应 `history/*_analysis.md`。禁止只更新 prediction 不更新 analysis。若预测覆盖多个彩种，相关彩种 analysis 必须全部更新到最新期号后，才可返回最终结论。

### 去核心化选号

大乐透、双色球、快乐八预测禁止直接围绕最新窗口 Top 热号骨架拼注。必须加入反集中约束（分区均衡、过热惩罚、热温冷混合），并在正文写明"已执行去核心化约束"。若结果明显由 Top 热号簇主导，必须执行降相关处理（过热号惩罚、分区上限）后再定稿。

### 大乐透/双色球重合约束

① 任一预测单式不得与历史任一期开奖完全重合；② 任一预测单式与最新一期开奖重合号数 ≤ 3（全号码位 7 码计算）。若不满足，必须替换低分重合号后再输出，并在正文标注"防重合约束已执行（历史不全重合 + 最新期≤3）"。

### 快乐八重合上限

预测"参考开奖 20 码"与最新一期真实开奖 20 码的重合数 ≤ 6。超出时按分数从低到高替换重合号，直至满足上限，并在输出中注明"重合约束已执行（≤6）"。

### 时间戳口径

`最后更新`、`预测生成时间` 统一使用**北京时间** ISO-8601（`+08:00`）。

### 组号金额带

统计规律输出完成后，配套至少一套**合计 10~30 元（含端点）/ 期**的投注推荐。不得仅以 2 元单注作为唯一收尾。常量 `DEFAULT_COMBO_BUDGET_MIN_YUAN=10`、`DEFAULT_COMBO_BUDGET_MAX_YUAN=30`。用户另有声明预算时从其声明。

### 单式优选强制输出

预测"明确号码"之后，必须额外给出 1 注分数最高的单式（大乐透前 5+后 2，双色球红 6+蓝 1，快乐八候选前 11），含总分、关键因子与生成时间。须保留随机性声明，不得表述为必中。

### 多因子加权规则（大乐透前区/双色球红球）

min-max 归一后按 8 项独立因子加权：**当前遗漏 22% + 频次 16% + 区间热度 15% + 近 K 期密度 13% + 奇偶对齐 9% + 大小/半区对齐 9% + 和值带对齐 4% + 马尔可夫转移 12%**（权重合计 1.0，见 `src/lottery/config.py`）。分区上限：大乐透前区/双色球红球每 5 码一小区至多 2 个；大乐透后区/双色球蓝球每 4 码一小区至多 2 个。快乐八 20 码按 8 个十码段**每段至少 1 个、至多 5 个**取满 20（先各段取最高分保底，再全局补满）；11 码在 20 码中按分降序抽，满足**每段至少 1 个、至多 5 个**。

### 禁止擅自创建或修改脚本/文件

- **禁止**在未经用户明确要求的情况下创建新的脚本文件（`src/scripts/*.py` 或其他任何位置）。
- **禁止**在未经用户明确要求的情况下修改现有脚本文件。
- **禁止**创建与项目核心目的（数据分析/预测/组号）无关的文档或辅助文件。
- 以上禁止适用于所有非用户主动请求的衍生文件、临时脚本、一次性工具等。

## AC 值（算术复杂度）

定义：号码排序后，所有两两差值去重个数 D，AC = D − (n−1)。


| 彩种  | 计算对象            | n   | 公式               |
| --- | --------------- | --- | ---------------- |
| 双色球 | 红球 6 枚          | 6   | AC = D − 5       |
| 大乐透 | 前区 5 枚          | 5   | AC = D − 4       |
| 快乐八 | 20 开奖号（用户明确要求时） | 20  | AC = D − 19（非主流） |


不适用：双色球蓝球仅 1 枚、大乐透后区仅 2 枚（n < 3）。

## 扩展统计分析维度（lottery-history-analysis 可用）

在频次、遗漏、奇偶、和值区间、共现、数据质量之外：

1. **时间维度聚合**：按自然月/年或滑动窗口，对每期衍生指标做均值、分位数、计数（须写清窗口定义）。
2. **期级衍生指标**：和值、跨度、连号个数、AC 值、质合比等（须写定义）。
3. **多窗口对照**：同一指标在 N₁/N₂/N₃ 期（如 30/100/300）下的对比表，仅作稳定性描述，不作「未来走势」结论。
4. **跨彩种对比**（可选）：仅允许表格化并列描述（期均和值、奇偶结构等），禁止推断因果或「联动开奖」。
5. **与随机基准对照**（可选）：在明确假设（号码独立均匀）下用解析或模拟得到参考分布对比观测分位；须声明假设与真实摇奖的差异，禁止包装为显著性「预测信号」。

---

## 扩展预测参考类型（lottery-prediction 可用）

1. **多窗口 N 对比**：同一指标在近 N₁ vs N₂ vs N₃ 的结论差异表；须说明窗口选择理由与结论不可外推。
2. **分位与区间**：和值、跨度等落在预设区间的历史频率；输出为频率叙述，不得改写为「下期必落某区间」。
3. **用户约束下的频次表**：在用户给定禁号、必选号、奇偶偏好等硬约束下的计数/频次叙述。
4. **组合结构参考**（非组号方案）：奇偶比、大小比历史频率，不与投注单混写。

## 预测参考归档


| 彩种  | 文件                                  |
| --- | ----------------------------------- |
| 大乐透 | `history/daletou_prediction.md`     |
| 双色球 | `history/shuangseqiu_prediction.md` |
| 快乐八 | `history/kuaileba_prediction.md`    |


每完成预测任务须更新对应文件，**必须**包含以下章节（顺序可微调，不得缺省）：

1. **口径说明**：彩种、期号范围、指标定义、权重说明
2. **结果摘要**：热号/冷号/结构（标注"已执行去核心化约束"）
3. **明确号码输出（强制）**：
  - 大乐透：5 注单式（前 5+后 2），每注逐号选择原因 + 预测生成时间
  - 双色球：5 注单式（红 6+蓝 1），每注逐号选择原因 + 预测生成时间
  - 快乐八：参考开奖 20 码（升序）+ 选十参考 11 码（升序），标注"重合约束已执行（≤6）"
  - 号码生成规则说明（多因子加权、分区上限、同分随机、惩罚机制）
4. **单式优选（强制）**：1 注分数最高的单式，含总分、关键因子贡献、生成时间
5. **使用说明**：随机性声明、娱乐用途说明
6. **附录：预算与投注推荐**：10~30 元金额带说明

## 职责隔离

本项目有 6 个 Skill（通过 `/skill-name` 调用）/ 6 个 Agent（同名），各司其职：


| Agent / Skill              | 专属职能                               | 禁止                        |
| -------------------------- | ---------------------------------- | ------------------------- |
| `lottery-manager`          | 意图解析、数据盘点、移交清单与建议提示词、跨阶段汇总（不含专精计算） | 统计报表、预测结论、采集落盘、投注方案       |
| `lottery-history-analysis` | 描述性统计、频次遗漏、数据质量报告                  | 预测推荐、采集落盘、投注组合、全流程编排      |
| `lottery-prediction`       | 冷热号/结构参考、明确号码输出、随机性声明              | 完整 EDA 替代、采集落盘、投注组合、全流程编排 |
| `lottery-draw-sync`        | 三彩种数据采集、解析、校验、落盘与溯源                | 统计分析、预测推荐、投注组合、全流程编排      |
| `lottery-draw-dlt-ssq`     | 仅大乐透+双色球数据同步                       | 快乐八同步、统计分析、预测推荐、投注组合      |
| `lottery-combo-optimize`   | 10~30 元内投注组合/注数/金额方案               | 历史 EDA、预测推荐、采集落盘、全流程编排    |


每个 Agent 独立运行，只完成本角色专属工作。禁止越界输出其他 Agent 的专属交付物。详细分工见 `AGENTS.md` 与 `.cursor/agents/*.md`。

### 组号 (combo-optimize) 硬约束

- 单期合计 10~30 元（含端点）。用户另有声明时从其声明。
- 大乐透/双色球：须覆盖单式、复式、胆拖的注数与金额计算。用户声明不要倍投时，仅用多注单式/复式/胆拖凑额，不得用倍数投注凑额。
- 快乐八：仅选十；复式仅允许 11 码选十复式（C(11,10)=11 注=22 元）。不得默认加入其他复式档位。
- 组号时不得重算统计预测；若要对齐预测结论，须引用 `history/*_prediction.md` 或用户粘贴的预测摘要作为输入。

## 推荐工作流

```
/lottery-manager（盘点 + 移交清单）
    → /lottery-draw-sync 或 /lottery-draw-dlt-ssq（补数据）
    → validate（python src/scripts/lottery.py validate）
    → /lottery-history-analysis（分析 + 更新 history/*_analysis.md）
    → /lottery-prediction（预测 + 更新 history/*_prediction.md）
    → /lottery-combo-optimize（可选组号）
```

---

## 异常数据闭环

历史分析发现异常 → `/lottery-draw-sync` 修正源或重建 processed → `/lottery-history-analysis` 重跑 → 按需 `/lottery-prediction` 重跑。
