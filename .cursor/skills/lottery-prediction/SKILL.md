# 统计型号码参考（非预测科学）

## 原则

- 所有输出均为 **历史描述在当期之前的投影**，不具有可证伪的未来预测效力。
- 开篇或结尾必须用一两句话声明随机性与娱乐用途（与 `lottery-core` 规则一致）。
- **去核心化约束（强制）**：大乐透、双色球、快乐八预测时，禁止直接围绕“最新窗口高核心组合（Top 热号骨架）”出号；必须在同一评分口径下加入反集中处理（如过热惩罚、分区均衡、热温冷混合），并在结果中明确已执行。
- **大乐透/双色球防重合（强制）**：每注预测单式不得与历史任一期开奖完全一致，且与最新一期开奖重合号码数必须 **<=3**；若超限需替换低分重合号后再输出并注明已执行。
- **时间戳规则（强制）**：`最后更新`、`预测生成时间` 使用北京时间 ISO-8601（`+08:00`）。

## 数据来源优先级（推荐）

1. **`data/processed/`** 规范化表（若存在）：大乐透 `dlt_draws.csv`、双色球 `ssq_draws.csv`，并参考 `manifest.json`。  
2. 否则：用户指定路径（本仓库不约定 `data/*.xlsx`）。  
3. 元数据中须写明路径；可引用 `history/*_analysis.md` 作为辅助说明，但**统计计算仍须基于原始表或 processed 之一可追溯复现**。

## 可采用的统计口径（需显式写明）

- **频次类**：近 N 期（N 由用户指定；**未指定时本仓库默认 N=30**，与 `src/lottery/config.py` 中 `DEFAULT_STATS_WINDOW` 及 `.cursor/rules/lottery-core.mdc`「默认统计窗口」一致；若采用 50/100/300 等多窗口须在文中逐项写明）。  
- **遗漏类**：当前遗漏期数、最大历史遗漏分位数（仅描述过去）。  
- **结构类**（视彩种）：奇偶比、大小比、和值区间出现频率；若含 **AC 值**分布或区间频率，定义与适用与 `.cursor/rules/lottery-stats-prediction-extended.mdc` 中「AC 值」节一致（双色球仅红球 6、大乐透仅前区 5）。  
- **快乐八选十视角**：若基于「开奖 20 码」做参考，须注明与「自选 10 码中奖」逻辑不同，避免混为一谈。

## 扩展统计参考口径（与 `lottery-stats-prediction-extended` 规则一致）

在用户需要时，可增加：

- **多窗口 N 对比**：如 N₁/N₂/N₃ 下同一结论表；写明各 N 与**不可外推**。  
- **分位与区间**：和值、跨度等落区历史频率；仅频率叙述，禁止「下期必落某区间」。  
- **用户约束下频次**：禁号、必选、奇偶偏好等约束下对剩余号码或合法结构的计数/排序；仍须随机性声明。  
- **组合结构参考**：红球/前区奇偶比、大小比等频率。  
- **与 `lottery-combo-optimize` 的边界**：本 Skill **必须**输出下文「明确号码输出」中的**完整可见号码**；**多注组合、10～30 元预算带内的复式注数分解、胆拖方案**仍属 `lottery-combo-optimize`。

## 输出结构模板

```markdown
## 口径说明
- 彩种：
- 期号范围：
- 指标定义：

## 结果摘要
（列表：热号 / 冷号 / 其他）

## 明确号码输出（强制）
> 一两句说明合成规则（如多因子加权：遗漏 > 频次 > 区间热度、与 combo 专精的边界）。

- **大乐透**：**5 注单式**，每注前区 5 + 后区 2；须写 **预测生成时间** 与 **每一号码的选择原因**（`regenerate-history` 脚本固写）。
- **双色球**：**5 注单式**，每注红 6 + 蓝 1；须写 **预测生成时间** 与 **每一号码的选择原因**（脚本固写）。
- **快乐八**：须含 **参考开奖 20 码**（升序，01–80）及 **选十参考 11 码**（升序）。仓库脚本口径：20 码 = 窗口内 **多因子** min-max 归一后，对 **当前遗漏、频次、区间热度** 加权（**遗漏 > 频次 > 区间热度**）再与其余因子合成，按 **8 个十码段（01–10,…,71–80）每段至少 1 个、至多 5 个** 取满 20，**同分随机**（**不是**从最后一期已开 20 码抽样，**不是**纯频次 Top20）；大乐透/双色球 **5 注** 为**同一加权 + 同分随机**：前区/红球 **每连续 5 个号一小区、每小区至多 2 个**；后区/蓝球 **每连续 4 个号一小区、每小区至多 2 个**（蓝仅 1 码时自然满足）；注间带**已选号惩罚**以拉开互异组合；11 码 = 优先在 20 码中**无放回随机抽取并校验分区约束**，若多次尝试仍不满足则按回退规则补齐（每次运行可不同）。须声明与真实开奖、选十中奖规则之区别。
- **去核心化落盘说明（强制）**：在「结果摘要」或「明确号码输出」中补一句“已执行去核心化约束”，说明本次未直接采用最新窗口 Top 热号骨架。

## 使用说明
以上仅为历史统计，下一期开奖仍为独立随机事件。
```

## 预测结果落盘（强制，每次任务）

受 `.cursor/rules/lottery-prediction-storage.mdc` 约束：每完成一次（或每个涉及彩种）统计型预测参考，须在**同一轮对话内**更新：

| 彩种 | 文件 |
|------|------|
| 大乐透 | `history/daletou_prediction.md` |
| 双色球 | `history/shuangseqiu_prediction.md` |
| 快乐八 | `history/kuaileba_prediction.md` |

**元数据块**须含：`最后更新`（ISO 8601）、`统计窗口`（如近 N 期）、`期号范围`、`所用数据路径`、可选 `引用分析归档路径`。正文须含「输出结构模板」全部章节，**且「## 明确号码输出（强制）」不可省略**（无号码视为未完成落盘）。**归档文件不可漏更**。

批量从 processed 重写预测正文（**含**机械「明确号码输出」：大乐透/双色球各 **5 注单式** + 逐号原因 + 预测时间；快乐八另含「参考开奖 20 码」；**默认近 30 期**）**统一**运行：`python src/scripts/lottery.py regenerate-history --only all`（存在 `kl8_draws.csv` 时才会覆盖三彩种预测）；仅快乐八：`--only kl8`；仅大乐透/双色球：`--only dlt-ssq`。**会覆盖**对应 `history/*_prediction.md` 全文。文末「**附录：预算与投注推荐**」：金额带 **10～30 元/期（含）**；大乐透/双色球说明正文 **5×2=10 元** 单式；快乐八为 **11 码复式**约 22 元；常量 `DEFAULT_COMBO_BUDGET_MIN_YUAN` / `DEFAULT_COMBO_BUDGET_MAX_YUAN`。若另有 **`lottery-combo-optimize` 更丰富打票表**附录，重跑后须补回。若需与一键结果不同的口径，由本 Agent 手写覆盖。

## 职责隔离（禁止越界）

本 Skill 仅服务 **`lottery-prediction`**。**禁止**：完整历史 EDA 替代报告（属 `lottery-history-analysis`）；抓取、解析、校验、落盘（属 `lottery-draw-sync`）；10～30 元内投注组合（属 `lottery-combo-optimize`）；全流程编排与移交清单（属 `lottery-manager`）。  
若缺数据或口径，可要求用户提供 **数据路径** 或 **`lottery-history-analysis` 的摘要**，不得代为重做全套历史分析。

## 禁止

- 不得输出「建议投入金额占收入比例」类财务建议。  
- 不得伪造「模型回测收益率」。
