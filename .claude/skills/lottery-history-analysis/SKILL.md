# 历史数据分析（大乐透 / 双色球 / 快乐八）

## 职责

做频次、遗漏、区间、奇偶等**描述性统计**；先做数据质量检查（缺期、重期、越界、重复行）。

## 数据来源优先级

1. **`data/processed/`** 下规范化文件优先使用：
   - 大乐透：`data/processed/dlt_draws.csv`
   - 双色球：`data/processed/ssq_draws.csv`
   - 快乐八：`data/processed/kl8_draws.csv`
   - 元数据：`data/processed/manifest.json`
2. 否则：`data/raw/` 或用户指定路径。
3. 输出元数据中写明实际采用的层级与路径。
4. 开始前可建议执行 `python src/scripts/lottery.py validate` 作为数据质量门禁。
5. 若仅需机械分析同步到 history：使用 `python src/scripts/lottery.py regenerate-history --only ...`。

## 默认统计窗口

用户未指定 N 时，三彩种默认**期末尾连续 30 期**（与 `DEFAULT_STATS_WINDOW` 一致）。

## 推荐分析维度

- **频次与遗漏**：每个号码出现次数、平均遗漏、当前遗漏。
- **分区**：大小号、奇偶、和值区间。
- **和值/跨度/AC 值/连号**：按彩种适用（AC 值定义见 CLAUDE.md）。
- **结构**：奇偶比、大小比、连号占比。
- **数据质量**：期号连续性、排序、重复行。
- **扩展维度**（用户需要时）：时间聚合、多窗口对照、随机基准对照。

## 快乐八特别说明

- 区分「每期 20 个开奖号」与「选十投注逻辑」；报告中写清统计视角。
- 频次/遗漏针对开奖 20 码的出现，不等同选十中奖概率。

## 输出格式

1. **摘要**：数据起止期、总行数、缺失与异常处理。
2. **分彩种结果**：表格或结构化说明。
3. **局限**：历史分布不蕴含可验证的预测能力。

## 归档（强制）

每次任务须更新对应文件（整文件覆盖）：

| 彩种 | 文件 |
|------|------|
| 大乐透 | `history/daletou_analysis.md` |
| 双色球 | `history/shuangseqiu_analysis.md` |
| 快乐八 | `history/kuaileba_analysis.md` |

元数据块须含：最后更新（北京时间 ISO-8601）、期号范围、所用数据路径。

## 职责隔离

**禁止**：预测类号码推荐、采集/落盘、投注组合、全流程编排。

## 反模式

- 不要用「机器学习准确率」包装纯随机过程。
- 不要将快乐八开奖 20 码等同于选十投注结果分析。
