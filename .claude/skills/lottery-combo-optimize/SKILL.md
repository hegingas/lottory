# 10～30 元投注组合优化（默认金额带）

## 全局约束

- **总预算（默认金额带）**：单期合计须在 **10～30 元（含端点）** 内，除非用户任务中**另行声明**其他预算；多期任务须说明每期拆分且各期仍建议落在用户声明的带内。  
- **与预测衔接**：统计型预测归档定稿后，面向打票的方案**不得**仅以 **2 元**单注作为唯一交付（除非用户明确放弃金额带要求）；须在 **10～30 元** 内给出至少一套可复核方案。  
- **大乐透、双色球**：须能处理 **单式、复式、胆拖** 的注数与金额；**倍投**仅在用户明确要求时使用。组合可为混合方案（例如一张复式 + 若干单式）。  
- **快乐八**：**仅选十**；复式 **仅允许 11 个号码的选十复式**（即从 11 码中取全部 C(11,10)=11 注，典型 **22 元**，已落在默认金额带内）。不得默认加入其他档位的快乐八复式，除非用户明确放开。
- **排列5**：每期 **5 位**（`d1`–`d5`，每位 **0–9**，**位间独立、允许重复**）。默认以 **单式多注** 落在 **10～30 元** 内（与 `lottery-prediction-storage` 一致时，可参考 **`history/pailie5_prediction.md` 附录**：**5 注单式 ≈10 元**，单价以官方为准）。用户明确要求时，可给出 **直选复式**（各位所选号码个数之**积**为注数）、**组选/倍投**等，但必须写明 **玩法全称 + 注数公式 + 小计**，且合计仍须满足用户声明或默认金额带。

## 注数速算（实现与手算校验用）

- **单式**：大乐透 1 注 = 5+2；双色球 1 注 = 6+1；各 **2 元/注**（以当期官方为准）。  
- **复式注数**：  
  - 大乐透：C(前区 n,5)×C(后区 m,2)  
  - 双色球：C(红 n,6)×C(蓝 m,1)  
- **胆拖**：前区/红球为「胆必中 + 拖中补齐」，后区/蓝球同理按组合计数；胆码数量必须小于玩法所需个数。  
- **快乐八选十 11 码复式**：注数 = C(11,10) = **11 注**，金额 = 11×**2 元** = **22 元**（单价以官方为准）。
- **排列5**：  
  - **单式**：1 注 = 一组完整 5 位号码；通常 **2 元/注**（以当期官方为准）。*N* 注单式金额 = *N*×单价。  
  - **直选复式（常见）**：若第 *i* 位从 *kᵢ* 个候选中选 1 个（每位各选 1 个），注数 = **k₁×k₂×k₃×k₄×k₅**；再乘以单价得金额。组选类玩法须按**当期规则**单独计数，不得套用大乐透复式公式。

## 「优化」目标函数（需与用户对齐其一或加权）

默认若无特别说明，采用 **透明可解释** 目标，例如：

1. **覆盖度**：在预算内最大化覆盖用户关心的号码集合或区间。  
2. **分散度**：降低单注完全重复结构（若用户在意「多样化」）。  
3. **约束满足**：胆码必选、禁号集、奇偶比软约束等。

说明：**不存在**期望收益最大化的诚实数学解（独立同分布假设下负期望）；优化的是用户给定偏好下的组合与风险控制感（仍须声明娱乐性质）。

## 输出清单

- **投注原因（强制）**：在给出任何具体注单或金额前，**必须**用简短文字说明**为何这样选**（可追溯、可复核），至少包含：  
  - **目标函数**：例如覆盖 `history/*_prediction.md` 中近 N 期热号、补足冷号/遗漏、胆拖约束、用户指定禁号等（择一或组合）；  
  - **依据来源**：若引用预测/分析归档，写明**文件路径与章节**（或用户粘贴的摘要）；若仅按用户主观号码组号，写明「未引用预测，按用户给定列表」。  
  禁止只列号码而不写原因。  
- 每张票：玩法类型（单式/复式/胆拖/快乐八选十11码/**排列5 单式或直选复式等**）、号码列表（排列5 建议按 `d1–d5` 分列或标准 5 位串）、注数、单价、小计。  
- **合计金额** 须在默认 **10～30 元（含）** 区间内（用户另有声明时从其声明）。  
- **校验**：逐条用公式复核注数。

## 职责隔离（禁止越界）

本 Skill 仅服务 **`lottery-combo-optimize`**。**禁止**：历史 EDA 与数据质量报告（属 `lottery-history-analysis`）；冷热/常见号预测（属 `lottery-prediction`）；开奖数据采集与落盘（属 `lottery-draw-sync`）；全流程编排（属 `lottery-manager`）。

## 与预测 / 分析的衔接（强制）

- **不得**在本对话中重新计算或「顺便给出」冷热、常见号等统计预测结论。  
- 若用户希望组号与某次统计参考对齐：须让用户提供 **`history/*_prediction.md` 路径**（排列5 对应 **`history/pailie5_prediction.md`**）或**粘贴预测摘要**（须含 **「明确号码输出」** 节，便于与 10～30 元方案对照），以及预算与玩法约束；本 Agent 仅在给定约束下构造注单与金额。  
- 若仅基于用户主观号码列表组号，须在输出中声明**未**引用预测归档。

## 反模式

- 不要把快乐八优化扩展到选九、选八等，除非用户明确要求。  
- 不要省略胆拖可行性（胆码个数非法、后区胆拖不合法等）。
- 不要把排列5 写成「前区/后区」球区复式；排列5 是 **5 位数字位**，复式注数为各位方案数之积（或按组选规则），与大乐透/双色球不是同一套计数模型。
